# 云端自跑部署指南 —— 彻底脱离 WorkBuddy

目标：让情报站**每天 9:00 在 GitHub 云端自己跑完、推到你手机**。
你的电脑可以关机，WorkBuddy 可以不开——它跑在 GitHub Actions 上，免费。

---

## 一、整体链路（跑在云端）

```
GitHub Actions 定时器（每天 9:00）
  → fetch_sources.py  抓网页源 + 预筛（不花钱）
  → denoise.py        调 DeepSeek 压成卡片（几分钱）
  → run.py 写 data.js + publish.py 归档
  → 提交回仓库 → GitHub Pages 自动发布（手机能打开的网址）
  → notify.py 弹 Bark 锁屏推送 → 点开看当天日报
```

> **X 不上云**：X 抓取要带登录态，不适合放云端。X 留在本机每周一/周四抓一次，
> 抓完把 `data.js` 里的 x 卡片同步到云仓库即可（见第五节）。云端只全自动跑网页源。

## 三、一次性配置（约 20 分钟）

### 第 1 步：拿 DeepSeek API key

1. 打开 https://platform.deepseek.com → 注册/登录
2. 左侧「API keys」→「创建 API key」→ 复制（只显示一次，存好）
3. 充值几块钱（一天一份日报约几分钱，几块钱够用几个月）

### 第 2 步：建 GitHub 仓库并开 Pages

1. https://github.com/new 建一个仓库，比如叫 `daily-intel`，选 **Public**（公开仓库 Actions 完全免费）
2. 把 `daily-intel/` 整个文件夹的内容传上去（用 GitHub Desktop 拖拽，或 `git push`）
   - **注意**：`x-userdata/`（X 登录态）**不要上传**，加进 `.gitignore`
3. 仓库 **Settings → Pages** → Source 选 `Deploy from a branch` → 分支选 `main` / 目录 `/ (root)` → Save
4. 等 1 分钟，页面会显示你的网址，形如 `https://你的用户名.github.io/daily-intel/`
   - 日报最新一期固定在 `https://你的用户名.github.io/daily-intel/today.html`

### 第 3 步：填 Secrets（加密保存 key，不会泄露）

仓库 **Settings → Secrets and variables → Actions → New repository secret**，加三个：

| Name | Value |
|---|---|
| `DEEPSEEK_API_KEY` | 第 1 步拿到的 key |
| `BARK_KEY` | iPhone 装 Bark App 后它给你的那串 key（见 `手机推送配置指南.md`） |
| `PAGE_URL` | `https://你的用户名.github.io/daily-intel/today.html` |

> 不填 `BARK_KEY` 也能跑，只是不推手机；不填 `PAGE_URL` 推送点开不跳转。

### 第 4 步：开启 Actions

1. 仓库 **Actions** 标签页 → 如果提示，点 `I understand my workflows, enable them`
2. 左侧选「每日情报站日报」→ 右上「Run workflow」**手动先跑一次**验证
3. 跑完看 `today.html` 有没有更新、手机有没有收到推送

成功后，**之后每天 9:00 它自己跑，你什么都不用做。**

## 五、X 数据怎么同步上云（每周一/周四）

云端不抓 X，所以 X 卡片靠你本机更新后推上去：

1. 本机周一/周四跑 `x_scrape.py`（沙箱外），得到新 X 数据
2. 手动把新的 x 卡片填进本机 `data.js`（或让我帮你跑一次）
3. `git add data.js && git commit -m "update x" && git push`

> 云端 `run.py` 每天会**原样保留** `data.js` 里现存的 x 卡片，只更新网页源那些频道。
> 所以你推一次 X，云端会一直沿用到你下次推新的。

## 六、和 WorkBuddy 的关系

配好这套后：
- **定时**：交给 GitHub Actions，不再依赖 WorkBuddy 自动化
- **降噪**：交给 DeepSeek，不再烧 WorkBuddy token
- **WorkBuddy 的角色**：只在你想「临时补跑 / 改样式 / 调规则」时用，日常零参与

可以保留 WorkBuddy 那个每日自动化作为**备份**（万一云端某天挂了），也可以直接关掉它。
关掉方法：让我帮你 `automation` 设为 PAUSED 即可。
